1. **Update `src/lib/convex.ts`:** Add `skipConvexDeploymentUrlCheck: true` option to the ConvexReactClient constructor. This is required for self-hosted Convex with custom domains. Keep the existing dev mode fallback logic intact.

```typescript
export const convex = new ConvexReactClient(url, {
  skipConvexDeploymentUrlCheck: true,
});
```

2. **Create `.env.production`:** Add production environment variables:
```
VITE_CONVEX_URL=<backend_url_from_plan_01>
```
Use the actual Convex backend URL provided by the user in Plan 01's checkpoint.

3. **Deploy Convex functions** to self-hosted instance:
```bash
CONVEX_SELF_HOSTED_URL='<backend_url>' \
CONVEX_SELF_HOSTED_ADMIN_KEY='<admin_key>' \
npx convex deploy
```
This pushes all schema definitions, queries, and mutations from the `convex/` directory to the self-hosted backend.
