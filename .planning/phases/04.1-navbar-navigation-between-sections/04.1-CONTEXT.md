# Phase 4.1: Navbar Navigation Between Sections - Context

**Gathered:** 2026-02-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a persistent, responsive navbar that gives users a way to navigate between sections of the app appropriate to their role(s). Currently there is no navigation — users are hard-redirected to a single section and cannot switch manually. This phase introduces the nav shell only (layout + routing); new sections/pages are not in scope.

</domain>

<decisions>
## Implementation Decisions

### Nav placement & style (responsive)
- **Mobile:** Fixed bottom tab bar with icon + label per tab
- **Desktop:** Always-expanded left sidebar (icon + label, no collapse toggle)
- **Mobile top bar:** Slim header above content showing current section name (page title)
- **Desktop top header:** Keep the existing header (title + SyncIndicator) on desktop; hide it on mobile
- Nav item visual style: Claude's discretion — match existing shadcn/ui aesthetic (active item highlighted)

### SyncIndicator & OfflineBanner placement
- Claude's discretion — integrate cleanly without cluttering the nav
- Current behavior (OfflineBanner at top, SyncIndicator in header) is the reference; adapt to new layout

### Multi-role visibility
- Show all sections the user has access to (not filtered to primary role)
- Nav is consistent across all pages — does not adapt based on current section context
- Only show nav items for sections the user can actually access (hidden if no access, not grayed out)
- Admin section takes priority on first login / default route
- No-role users see a minimal nav with "Contact your admin to get access"

### Sections per role
- **Admin:** Form Builder + Fill Forms + Reviewer Dashboard (sees all three for testing)
- **Worker:** Fill Forms only
- **Reviewer:** Reviewer Dashboard only
- Users with multiple roles see all sections they have access to
- Nav labels and icons: Claude's discretion (Lucide icons, short clear labels)

### User identity
- Show avatar (or initials fallback) + name from Clerk user object (firstName, lastName, imageUrl)
- **Desktop:** User identity at the bottom of the sidebar
- **Mobile:** User identity in the top bar
- No sign-out button in this phase

</decisions>

<specifics>
## Specific Ideas

- Mobile layout: bottom tabs (icon + label) + slim top bar showing page name + user avatar in top bar
- Desktop layout: always-expanded left sidebar with nav items + user identity footer, existing top header retained
- Admin gets all three sections in nav (Builder, Fill Forms, Dashboard) for testing purposes
- Clerk provides the avatar image and name — use `imageUrl`, `firstName`, `lastName`

</specifics>

<deferred>
## Deferred Ideas

- Sign-out button in the nav — future phase (settings/profile page)
- Collapsible sidebar (icon-only mode) — future enhancement
- "Most recently visited section" restore on login — future enhancement
- Admin read-only submissions view as a separate nav item — future phase

</deferred>
