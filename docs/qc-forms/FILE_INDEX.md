## 📊 Key QC Parameters Across All Forms

All forms track these core metrics:
- **Temperature**: Receiving, storage, transportation
- **Weight/Glaze**: Calculated net weight, glaze percentage
- **Appearance**: Color, texture, visual defects
- **Odor/Flavor**: Sensory quality indicators
- **Count/Size**: Species-specific grading standards
- **Packaging**: Integrity, labeling, compliance
- **Defects**: Documentation and photography
- **Traceability**: Lot codes, dates, supplier info

## 💡 Implementation Guidance for Nexus

### Schema Design Reference
Use these forms to define QC data fields:
- Product type (species)
- Temperature readings
- Packaging condition
- Visual quality assessments
- Defect documentation
- Organoleptic evaluations
- Approval decision

### Form Generator Configuration
Build dynamic forms based on:
- Species-selected → load species-specific fields
- Critical limits → enforce validation rules
- Operator role → show relevant inspection procedures
- Supervisor role → display approval criteria

### Approval Workflow
Implement supervisor checklist against:
- Sensory quality standards
- Critical control point compliance
- Regulatory requirements
- Defect acceptance thresholds

### Audit Trail
Document per FDA requirements:
- Inspector identification
- Date/time of inspection
- Temperature records
- Lot traceability
- Approval decision and rationale
- Any deviations or corrective actions
