## Gap Details

### NEW: Mobile Sync Worker Never Starts

**File:** `src/App.tsx` + `src/components/sync/SyncIndicator.tsx` + `src/db/sync/worker.ts`

The background sync loop (`startSyncWorker`) is only called inside `useSync()`, which is only consumed by `SyncIndicator`. `SyncIndicator` is only rendered in `AppLayout`'s desktop header. On mobile devices, `BottomTabBar` has no SyncIndicator, so `startSyncWorker` is never called.

**Effect:** Mobile workers rely solely on `triggerSync()` called once after each submission (`FormFillingPage.tsx:255`). If that one-time sync fails, submissions accumulate in IndexedDB with no background retry. Workers cannot see sync status on mobile.

**Affects:** OFFL-01, OFFL-02

---

### NEW: forceDevMode Bypasses Clerk in All Dev Builds

**File:** `src/App.tsx` lines 13–34

```typescript
const forceDevMode = import.meta.env.DEV; // always true in development
if (forceDevMode || isDevModeWithoutCredentials) {
  return <MockAuthProvider>...</MockAuthProvider>;
}
```

`window.Clerk` is undefined when `MockAuthProvider` is used. Sync worker's `getAuthToken()` (`worker.ts:22`) throws `"No Clerk session"` on every sync attempt in dev. Prevents E2E testing without a production build.

**Affects:** AUTH-01/02/03 in dev, OFFL-01, REVW-01/04 in dev
