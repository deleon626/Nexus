# Phase 04.1 Plan 01: Layout Shell Components Summary

**One-liner:** Responsive layout shell with desktop sidebar, mobile bottom tabs, and role-based navigation filtering.

---

## What Was Built

### Core Layout Shell

Created a complete responsive navigation shell with five new components under `src/components/layout/`:

1. **NavItem.tsx** — Shared nav item component with four exports:
   - `NavItem` component: icon + label link with active state highlighting via `useLocation()`
   - `useNavItems()` hook: role-based filtering of nav items (admin sees all, worker sees forms, reviewer sees dashboard)
   - `useUserIdentity()` hook: Clerk `useUser()` wrapper with dev mode guard
   - `UserAvatar` component: image fallback to initials

2. **Sidebar.tsx** — Desktop always-expanded left sidebar:
   - App logo/title at top
   - Nav items in scrollable middle area
   - "Contact your admin" message for no-role users
   - User identity footer with avatar + name

3. **BottomTabBar.tsx** — Mobile fixed bottom tab bar:
   - Icon-centered tabs with labels below
   - Active state highlighting
   - Returns null for no-role users

4. **MobileTopBar.tsx** — Mobile slim top bar:
   - Current section name derived from pathname
   - User avatar (size="sm") on the right

5. **AppLayout.tsx** — Main layout shell:
   - Flex container with sidebar + main area
   - `<Outlet />` for child routes (React Router v7 pattern)
   - Desktop header with title + SafeSyncIndicator
   - SafeSyncIndicator wrapper to prevent dev mode crashes

## Technical Details

### Role-Based Nav Filtering

The `useNavItems()` hook filters navigation items based on user role:

```typescript
const NAV_CONFIG = [
  { label: "Builder", path: "/admin/builder", icon: LayoutGrid, requiresAdmin: true },
  { label: "Fill Forms", path: "/worker/forms", icon: ClipboardList, requiresWorker: true },
  { label: "Reviews", path: "/reviewer/dashboard", icon: CheckSquare, requiresReviewer: true },
];
```

- Admin users see all three items (for testing)
- Worker users see only "Fill Forms"
- Reviewer users see only "Reviews"
- No-role users see "Contact your admin" message in sidebar

### Dev Mode Guard

The `useUserIdentity()` hook guards Clerk's `useUser()` behind `isDevModeWithoutCredentials` to prevent crashes when ClerkProvider is not mounted:

```typescript
if (isDevModeWithoutCredentials) {
  return { name: "Dev User", imageUrl: null };
}
const { user } = useUser();
```

### Active State Detection

Uses `useLocation()` from react-router for reactive active state:

```typescript
const isActive = pathname === path || pathname.startsWith(path + "/");
```

### Responsive Layout

- Desktop (md+): Sidebar visible, bottom tabs hidden, desktop header visible
- Mobile (< md): Sidebar hidden, bottom tabs fixed, mobile top bar visible, main content has `pb-16` to prevent obscuring by bottom tab bar
